回顧在建構函式主體中定義方法的問題​

什麼事建構函式( constructor )呢？​

簡單來說，建構函式又稱為構造函式就是一個 Object 的模板，在建構函式中描述及定義 Object 結構和屬性，並可以透過 new 關鍵字將 Object 實體化生產出來，也就是實例。

// 建構函式 Point 有 x 、y、 setTo() 、logMyData() 屬性
function Point(x = 0, y = 0) {
  this.x = x;
  this.y = y;

  this.setTo = function (x, y) {
    this.x = x;
    this.y = y;
  };

  this.logMyData = () => {
    console.log("x:", x, "y:", y);
  };
}

// 使用建構函式建立 Object 實例

const point1 = new Point(1, 2);
const point2 = new Point(10, 20);

// 不同實例出來的Object 都輸出不同的結果，每個實例出來的 Object 都有自己獨立的屬性
point1.logMyData(); // x: 1 y: 2  .
point2.logMyData(); // x: 10 y: 20.

而在 JavaScript 中建立實例最簡單且有效率的方法就是使用原型 ( prototype ) 來定義方法，而非直接在建構函式中直接定義方法，因為每當實例出來都是一個獨立的屬性，重複創建會佔記憶體。

什麼是 Prototpye​

介紹 Prototype 之前要先講物件導向程式語言分為兩大類

  1. 基於 class 類別的物件導向語言（例如：Java，C#，Python） 。
  2. 基於 Prototype 的物件導向語言（例如:JavaScript）。

而經典的 OOP 語言，類別會從其他類別繼承其屬性或是方法等。

而像 JavaScript 這種基於 Object 物件也存在繼承，從物件繼承其屬性和方法等。

內部[[Prototype]]屬性​

JavaScript 中、每一個物件內部都有這樣的一個內部屬性 [[Prototype]]，這個屬性保存了該物件對其原型的引用。 而這個 [[Prototype]] 這種表示方式只是讓 JavaScript 可以 Follow ECMAScript 的規範，實作上並不會使用到這部分。

const a = { x: 1, y: 2 };
const b = { p: 10, q: 20 };

假設 b 的原型是 a，那麼b內部會長類似這樣：

b: {
    p: 10,
    q: 20,
    [[Prototype]]: a
}

而 const testArr = [1,2,-5]，其內部會長類似這樣：

testArr:{
  0 : 1 ,
  1 : 2 ,
  2 : -5 ,
  length : 3 ,
  [[Prototype]] : Array.prototype
}

當我們用宣告變數的方式，建立一個 Object 時 const obj = {x:1,y:2} ，其內部會長類似這樣：

obj:{
x : 1 ,
y : 2 ,
[[Prototype]]: Object.prototype
}
